    django-mysql (OpenSource Contribution)

    https://django-mysql.readthedocs.io/en/latest/

    2020.06~

    참고: 내가 Open한 Issue(https://github.com/adamchainz/django-mysql/issues/671)

    이거는 최근에 작업중인 PullRequest, Django QuerySet중 Model.objects.filter().update(field1="blabla") 수행시 modified_at(수정날짜)가 갱신이 안되는데 Mysql UPDATE SQL 수행시 날짜를 알아서 갱신해주는 옵션을 줄 수 있다.
    Django는 이 옵션 제공안해주고 버전 릴리즈 주기가 길어서(6개월) new Feature의 경우 머지되더라도 마이너버전업 할때는 배포 안해줌...
    Django-Mysql에 mysql자체에서 제공하는 "수정날짜 자동갱신 옵션"(on update current_timestamp)을 줄 수 있게하는 기능을 추가하려고 작업중...

    대체로 오픈소스 팀들은 NewFeature를 제안하는 경우 매우 엄격하다는 것을 느낌 새로운 기능은 새로운 버그를 가져오기때문인 듯

  • 2019

    pytest-tipsi-django (OpenSource Contribution)

    https://pypi.org/project/pytest-tipsi-django/

    2019.10

    pytest Django Transaction Manager Library : django pytest 수행시 더미데이터를 매번 testcase마다 INSERT rollback 하는 작업을 반복함. Spring에서 @BeforeClass와 같이 testcase 마다 transaction을 제어해서 더미데이터 재사용함
    하지만 pytest-django는 testcase간 transaction Managing기능이 없음 pytest-tipsi-django는 이를 지원해주는 라이브러리,
    신규프로젝트(Vendor 관리 서비스 개발)에서 해당 라이브러리를 도입해서 사용중이였는데 버그를 발견해서 이를 수정함,

    두번째 PullRequest는 라이브러리 자체가 불안정한건지 몇몇 상황에서 transaction save_point를 찾지못하는 이슈가 발생함 이를 개선하기위해 반드시 transaction 구간을 설정해주도록 작업하자고 제안했는데 Organizer가 거절함


    * https://github.com/tipsi/pytest-tipsi-django/pull/9 (Merged)
    * https://github.com/tipsi/pytest-tipsi-django/pull/10 (Closed)
